# Mobile-Poser & Sensor Analysis Project

This repository contains tools and experiments for processing raw IMU data from smartphones and smartwatches, analyzing motion patterns, and generating 3D human poses using the MobilePoser model.

---

## Repository Structure

### `sensor-test-drive/`
*Focus: Data exploration and classical signal analysis.*
- **`data-structure-exploration.ipynb`**: Investigates CSV schemas, sampling rates, and column mapping.
- **`data-analysis-walking.ipynb`**: Comparative plots and signal features for walking vs. sitting.
- **`data-analysis-stairs.ipynb`**: Analysis of vertical movement signals (upstairs vs. downstairs).
- **`README.md`**: Summary of findings from the initial sensor analysis phase.

### `poser-test-drive/`
*Focus: 3D Pose estimation and visualization.*
- **`final-sensor.ipynb`**: Prepares raw recording data for the MobilePoser model.
    > **Note:** Requires MobilePoser source code and model weights to be installed locally.
- **`visualize_sensor.py`**: A utility script to visualize the SMPL mesh output generated by the model.
- **`README.md`**: Setup instructions and performance summaries.

### `data/`
*Focus: Raw and processed sensor recordings.*
- **`upstairs-downstairs/`**: Recordings of vertical transitions.
    - `default/`: Standard Android/iOS sensor output.
    - `sensor-zoo/`: High-frequency (100Hz) data recorded via Madgwick orientation and adaptive thresholding.
- **`walking-sitting/`**: Recordings of horizontal movement and stationary states.
    - `default/`: Includes subfolders for **`processed`** (MobilePoser-ready tensors) and **`unity_motion`** (outputs for Unity3D visualization).
    - `sensor-zoo/`: High-frequency algorithmic data.


---

## CLI Visualization Quick-Start
From the `poser-test-drive` directory:

```bash
# View pose with walking path translation
python visualize_sensor.py --npz ../data/walking-sitting/default/processed/step0_output.npz --with-tran

# Export for Unity
python visualize_sensor.py --npz [path_to_npz] --export-unity ./unity_output
```
